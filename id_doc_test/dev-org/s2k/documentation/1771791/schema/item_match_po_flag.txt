### Match PO (item_match_po_flag)

* **Description**: This field determines whether an item should match a Purchase Order (PO). It automatically sets to "No" if the item has a GL code that is not empty and not equal to '201000'. Otherwise, it sets to "Yes".

* **Field type**: formula

* **Data type**: enum (string options: "Yes" or "No")

* **Required**: False

* **Exported**: False

* **Hidden**: False
